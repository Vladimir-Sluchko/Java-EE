Использование библиотеки jackson, изучение различий GET и POST, изучение аннотаций из библитеки, изчение раличных настроек стратегий нейминга и сериализации\десериализации в jackson

1. Скачать dto https://www.dropbox.com/s/mgugf5peaoqz3hi/dto_for_jackson.zip?dl=0
2. Структуру (поля, конструкторы, методы) класса менять нельзя. Можно поменять адрес пакета, можно добавлять аннотации, регистрировать модули для настроек ObjectMapper.
2. Класс dto Student.
	1. Для JSON использовать стратегию нейминга LOWER_CAMEL_CASE
	2. Создать сервлет /students.
		GET - возвращает список ранее созданных студентов в формате json. Не HTML!
		POST - создание из json студента и добавление созданного студента в заранее созданный список List<Student>. Использовать Request Body
3. Класс dto Citizen.
	1. Для JSON использовать стратегию нейминга SNAKE_CASE
	2. Для поля birthday в классе Citizen в json отображать как строку
	3. Для поля createDate в классе Passport в json отображать как число дней от 01.01.1970 года
	4. Создать сервлет /сitizens.
		GET - возвращает список ранее созданных граждан в формате json. Не HTML!
		POST - создание из json гражданина и добавление созданного гражданина в заранее созданный список List<Citizen>. Использовать Request Body
